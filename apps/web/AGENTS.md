# AGENTS

This file defines working rules for changes inside `apps/web`.

## Primary Principles

- Preserve the existing visual system defined by `apps/web/ARCHITECTURE.md`.
- Prefer local consistency over one-off styling decisions.
- Do not introduce hard-coded visual colors unless the component already does so intentionally.
- Keep animations restrained and use `AnimationContainer` for staged reveal patterns.

## SEO Rules

- Use `apps/web/utils/metadata.ts` as the default metadata entry point.
- For static routes, export `metadata = createMetadata({...})` from the route `layout.tsx`.
- For dynamic routes, use `generateMetadata(...)` and derive values from route data.
- Always set a real `path` so canonical URLs resolve correctly.
- Only reference social images that actually exist under `apps/web/public` or are generated by route handlers.
- Add structured data only when it maps to a real entity:
  - `BlogPosting` for article pages
  - `Product` for product detail pages
  - `BreadcrumbList` when breadcrumb structure is visible and meaningful
  - `FAQPage`, `Service`, `ContactPage`, or `Organization` only when the page content supports it
- If a route should not be indexed, use `noIndex: true`.

## Crawler Assets

- Keep `apps/web/app/robots.ts` and `apps/web/app/sitemap.ts` current.
- Any new public route should be considered for sitemap inclusion.
- Any new dynamic collection should be added to sitemap generation if it is indexable.

## Accessibility Rules

- Do not nest interactive elements.
- Preserve heading hierarchy where possible.
- Keep focusable controls keyboard accessible.
- Respect reduced motion for major animations.

## Verification

Use these checks after meaningful app changes:

- `npm --prefix apps/web run check-types`
- `npm --prefix apps/web run build`

If build fails due to a locked `.next` file on Windows, clear the lock and rerun rather than changing code blindly.
